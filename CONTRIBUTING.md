# Contributing

By participating, you agree to the [Code of Conduct](CODE_OF_CONDUCT.md).

1. Open an issue for behavior changes or architectural changes before investing in a large implementation.
2. Use Java 21 and run `./gradlew check` before submitting a pull request.
3. Keep dependencies flowing in the direction documented in `docs/architecture.md`.
4. Add tests for observable behavior and update an ADR when a recorded decision changes.
5. Keep commits focused and use clear, imperative commit subjects.

Pull requests must explain the problem, the chosen approach, verification performed, and compatibility impact.
Do not include generated build output, credentials, server data, or third-party binaries.
